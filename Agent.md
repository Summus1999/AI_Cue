# AI_Cue Agent Harness 文档

> 本文档是 AI Agent 在 AI_Cue 项目中的操作指南和约束规范。
> 每次会话开始时，Agent 必须阅读此文档。
> 当 Agent 犯错时，人类工程师应更新此文档以改进约束。

---

## 1. 强制加载机制

本项目的 Cursor 规则 `.cursor/rules/agent-harness.mdc` 已配置 `alwaysApply: true`，确保每次会话都会加载本 harness 的加载指令。Agent 在响应任何用户请求之前，必须先读取本文档并遵循其约束。

每次开始新会话时，Agent 必须读取 `TODO.md` 查询当前项目进度，并据此开展后续工作。Agent 永远不得修改 `TODO.md`，该文件仅允许人类工程师修改。

`TODO.md` 中的进度标记约定：
- ✅ 表示该功能点已完成
- ❌ 表示该功能点未完成

## 3. Harness 约束

### 3.1 编码前流程

- 实现代码修改之前必须先给出方案，待用户选择方案后再进行编码
- 在用户明确要求编码之前，仅进行方案讨论，不编写代码

### 3.2 代码规范

- 代码简洁，遵循奥卡姆剃刀原则
- 代码注释一律使用中文
- 输出高质量代码，减少无意义的日志
- 日志默认不做脱敏处理

### 3.3 沟通规范

- 回答使用中文
- 输出循序渐进，每一步详细解释
- 内容深度要深，提供详细解释
- 语言风格严谨
- 有场景不确定时直接提问，不浪费时间和 token

### 3.4 文档规范

- 写 markdown 文件一律不加表示加粗的双 `*` 号
- 没有要求不许写文档
